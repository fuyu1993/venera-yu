## 项目简介

venera-yu 是一款跨平台漫画阅读器，支持阅读本地和网络漫画。基于 Flutter 框架开发，使用 Rust 实现高性能模块，通过 JavaScript 引擎支持可扩展的漫画源系统。

**主要特性：**

- 阅读本地漫画（支持目录和压缩包格式）
- 使用 JavaScript 创建和加载网络漫画源
- 收藏、下载漫画
- 查看评论、标签等漫画信息
- 支持账号登录、评论、评分（需漫画源支持）
- 多平台支持：macOS / Windows / Linux / Android / iOS
- 无头模式（Headless），支持命令行自动化

## 从源码构建

### 准备工作

```bash
# 1. 克隆仓库
git clone <repository-url>
cd venera-yu

# 2. 安装 Flutter（参考 https://flutter.dev/docs/get-started/install）

# 3. 安装 Rust（参考 https://rustup.rs/）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
```

### macOS

```bash
# 设置编码环境变量（CocoaPods 要求）
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8

# 确保 Rust 环境已加载
source "$HOME/.cargo/env"

# 构建 Release 版本
flutter build macos

# 产物位置：build/macos/Build/Products/Release/venera-yu.app

# Debug 模式运行
flutter run -d macos

# 启动构建好的应用
open build/macos/Build/Products/Release/venera-yu.app
```

**已知警告（不影响构建）：**

- 多个插件不支持 Swift Package Manager（`desktop_webview_window`、`flutter_inappwebview_macos`、`rhttp` 等）
- `AppDelegate.swift` 迁移提示

### Windows

```bash
# 构建
flutter build windows

# 产物位置：build/windows/x64/runner/Release/
```

### Linux

```bash
# 安装依赖（以 Debian/Ubuntu 为例）
sudo apt install libgtk-3-dev libwebkit2gtk-4.1-dev

# 构建
flutter build linux

# 产物位置：build/linux/x64/release/bundle/
```

### Android

```bash
# 构建 APK
flutter build apk

# 构建 App Bundle
flutter build appbundle

# 产物位置：build/app/outputs/flutter-apk/ 或 build/app/outputs/bundle/release/
```

### iOS

```bash
# 构建（需在 macOS 环境下）
flutter build ios

# 使用 Xcode 打开并签名
open ios/Runner.xcworkspace
```

## 核心功能

### 应用启动

首次启动时，应用会初始化本地数据库并加载漫画源列表。

### 漫画源管理

1. 打开应用后，进入「源」或「发现」页面
2. 点击「添加源」，输入漫画源仓库 URL
3. 推荐使用官方源仓库：
   ```
   https://cdn.jsdelivr.net/gh/venera-app/venera-configs@main/index.json
   ```
4. 选择需要安装的漫画源并确认

### 阅读漫画

- **浏览**：在发现页浏览热门/最新漫画
- **搜索**：使用搜索功能查找漫画
- **收藏**：点击收藏按钮将漫画加入书架
- **下载**：在漫画详情页下载章节以便离线阅读
- **阅读器**：支持缩放、翻页方向切换、亮度调节等功能

### 手势操作

- 左右滑动翻页
- 双击缩放
- 长按呼出菜单
- 点击屏幕中间显示/隐藏工具栏

## 漫画源配置

漫画源使用 JavaScript 编写，通过 `flutter_qjs` 引擎运行。

### 添加源仓库

1. 打开应用 → 源管理 → 添加仓库
2. 输入 JSON 索引文件的 URL
3. 索引文件格式：

```json
[
  {
    "name": "源名称",
    "url": "https://example.com/source.js",
    "filename": "相对路径",
    "version": "1.0.0",
    "description": "源描述"
  }
]
```

## 本地漫画导入

### 支持的格式

| 格式  | 说明                     |
| ----- | ------------------------ |
| 目录  | 包含图片文件的文件夹       |
| .cbz  | ZIP 格式漫画压缩包         |
| .cb7  | 7z 格式漫画压缩包          |
| .zip  | 标准 ZIP 压缩包            |
| .7z   | 标准 7z 压缩包             |

### 目录结构要求

**无章节模式：**

```
漫画目录/
├── cover.png       # 封面（可选）
├── 001.jpg
├── 002.jpg
└── ...
```

**有章节模式：**

```
漫画目录/
├── cover.png       # 封面（可选）
├── 第1话/
│   ├── 001.jpg
│   └── 002.jpg
└── 第2话/
    ├── 001.jpg
    └── 002.jpg
```

### 导入步骤

1. 打开应用 → 本地 → 导入
2. 选择漫画目录或压缩包
3. 应用自动识别并导入
4. 导入后可在「本地」标签页查看

### 恢复本地下载

如果迁移应用后丢失了 `local.db` 数据库：

1. 确保设置中的本地存储路径指向包含已下载漫画的文件夹
2. 打开 `本地` → `导入` → `恢复本地下载`
3. 应用会扫描当前路径并重建条目

## 无头模式

无头模式允许从命令行执行自动化任务，适合与脚本或定时任务集成。

### 基本用法

```bash
venera-yu --headless <命令> [选项]
```

### 可用命令

| 命令                        | 说明                       |
| --------------------------- | -------------------------- |
| `webdav up`                | 上传配置到 WebDAV 服务器     |
| `webdav down`              | 从 WebDAV 下载配置           |
| `updatescript all`         | 更新所有漫画源脚本           |
| `updatesubscribe`          | 更新所有已订阅漫画           |
| `updatesubscribe --update-comic-by-id-type <id> <type>` | 更新指定漫画 |

### 示例

```bash
# 更新所有漫画源脚本
venera-yu --headless updatescript all

# 更新订阅的漫画
venera-yu --headless updatesubscribe

# 上传配置到 WebDAV
venera-yu --headless webdav up

# 抑制日志输出
venera-yu --headless --ignore-disheadless-log updatesubscribe
```

输出为 `[CLI PRINT]` 前缀的 JSON 格式，便于脚本解析。

详细说明参见 [无头模式文档](headless_doc.md)。

## 常见问题

### Q: macOS 构建失败？

1. 确保设置了编码环境变量：
   ```bash
   export LANG=en_US.UTF-8
   export LC_ALL=en_US.UTF-8
   ```
2. 确保 Rust 环境已加载：
   ```bash
   source "$HOME/.cargo/env"
   ```
3. 尝试清理后重新构建：
   ```bash
   flutter clean
   flutter pub get
   flutter build macos
   ```

### Q: 应用意外退出无法启动？

1. 删除本地数据并重新启动（注意备份）：
   ```bash
   rm -rf ~/Library/Application\ Support/venera-yu/
   ```
2. 如果问题持续，尝试清理 Flutter 缓存：
   ```bash
   flutter clean
   flutter pub get
   ```

### Q: 找不到漫画源？

检查源仓库 URL 是否有效，或在应用中手动刷新源列表。

### Q: 如何修改应用名称/图标？

- **名称**：修改 `macos/Runner/Configs/AppInfo.xcconfig` 中的 `PRODUCT_NAME`
- **图标**：替换 `macos/Runner/Assets.xcassets/AppIcon.appiconset/` 下的图标文件
- **版本号**：修改 `pubspec.yaml` 中的 `version` 和 `lib/foundation/app.dart` 中的 `version`

### Q: CocoaPods 版本过低？

```bash
sudo gem install cocoapods
pod --version  # 确认 >= 1.16.2
```

## 开发指南

1. `pubspec.yaml`：修改 `version: X.Y.Z+BUILD`
2. `lib/foundation/app.dart`：修改 `final version = "X.Y.Z"`

### 修改应用名

- **macOS**：`macos/Runner/Configs/AppInfo.xcconfig` → `PRODUCT_NAME`
- **Linux**：`linux/my_application.cc` → `gtk_window_set_title`

- **Android**：`android/app/src/main/AndroidManifest.xml`
